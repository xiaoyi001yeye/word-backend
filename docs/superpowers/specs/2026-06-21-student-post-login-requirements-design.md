# 学生登录后功能清单与详细需求设计

## 背景

当前系统已经具备用户登录、学生角色识别、学生可见词书、学习计划、今日任务、学习记录、注意力统计和考试作答等能力。学生端仍以词书浏览为主要工作台形态，今日学习能力通过模态窗进入，个人中心、自由学习、错词本、生词本、自测和分析型统计尚未形成完整闭环。

本文档用于定义学生登录后的现状能力、目标版功能清单和详细需求边界，作为后续学生端产品与技术实现计划的依据。

## 目标

- 学生登录后默认进入“今日任务优先”的学习工作台。
- 在老师布置的学习计划之外，允许学生对已分配词书进行自由学习。
- 建立“计划内进度 + 学生全局卡片盒”的两层记忆模型。
- 将自动错词本与手动收藏夹拆分展示，并都服务于个人复习。
- 区分老师正式考试和学生自测，避免自测污染老师评价数据。
- 提供最近 30 天学习日历和分析型统计。
- 提供学生可自维护的个人中心。
- 保持通知范围为首页轻提醒，不引入消息中心或外部推送。

## 非目标

- 不做排行榜、积分商城、班级动态等运营型功能。
- 不做家长端或家长可见报告。
- 不做短信、邮件、微信等外部提醒。
- 不允许学生学习未分配词书之外的全库词汇。
- 不允许学生修改用户名、角色、班级、老师归属。
- 不把学生自测结果计入老师正式考试评价。

## 当前能力对照

| 模块 | 当前能力 | 目标版变化 |
|------|----------|------------|
| 登录 | 支持用户名密码登录、JWT、`/api/auth/me` | 保留，学生登录后进入今日任务工作台 |
| 学生词书 | `GET /api/students/me/dictionaries` 查看已分配词书 | 保留，并支持从词书进入自由学习 |
| 单词浏览 | 可查看词书单词、单词详情、搜索词汇 | 保留，增加收藏、生词本和自学入口 |
| 今日学习 | `GET /api/students/me/study-plans`、`/{id}/today`、`/{id}/records` | 从模态窗能力升级为首页核心任务流 |
| 注意力统计 | 已记录停留时长、有效专注、空闲、交互次数 | 扩展到分析型统计趋势 |
| 考试 | 老师/管理员创建，学生作答和查看结果 | 新增学生自测，正式考试和自测分轨 |
| 错词 | 学习进度已有 `wrongTimes`，考试结果可识别错题 | 新增自动错词本，统一沉淀到全局卡片盒 |
| 收藏 | 当前无独立学生收藏能力 | 新增手动收藏夹 |
| 个人中心 | 可通过本人 ID 查看基础资料，暂无学生自改接口 | 新增资料编辑和密码修改 |
| 通知 | 当前无消息中心 | 首页轻提醒，不建独立消息表 |

## 设计原则

1. 今日任务优先：学生登录后的第一问题是“今天要做什么”。
2. 资源边界清晰：学生只能学习老师或系统分配给自己的词书。
3. 进度分层：计划内进度服务老师布置；全局卡片盒服务学生长期记忆。
4. 评价分轨：正式考试服务老师评价；自测服务学生练习。
5. 自动与手动并存：错词自动进入，收藏由学生主动管理。
6. 轻提醒优先：提醒来自任务、考试、复习积压的实时聚合，不先建设消息系统。

## 目标版功能模块

### 1. 登录与学生首页

#### 功能说明

学生登录成功后进入学生工作台。首页顶部展示今日学习任务摘要，并提供“继续学习”主按钮。词书、自测、统计、个人中心作为次级入口。

#### 首页内容

- 今日任务摘要：待学新词、今日复习、逾期复习、已完成数量、总任务数量。
- 主行动按钮：
  - 有未完成任务时显示“继续今日学习”。
  - 今日任务完成时显示“查看学习统计”或“自由学习”。
- 轻提醒：
  - 今日学习计划未完成。
  - 存在逾期复习。
  - 存在待完成正式考试。
- 快捷入口：
  - 我的词书。
  - 自测练习。
  - 错词本。
  - 生词收藏。
  - 学习统计。
  - 个人中心。

#### 权限规则

- 学生不能看到管理端入口。
- 学生不能看到创建词书、导入词书、用户管理、班级管理、学习计划管理等管理按钮。
- 管理员登录用户端时仍按现有逻辑跳转管理端。

#### 验收标准

- 学生登录后默认首页以今日任务为首屏主内容。
- 学生无学习计划时，首页展示已分配词书和自由学习入口。
- 存在逾期复习时，逾期数量明确可见。
- 学生看不到任何管理动作入口。

### 2. 今日学习

#### 功能说明

今日学习承接老师发布的学习计划。学生点击“继续今日学习”后进入逐词学习流程。

#### 队列规则

今日队列优先级固定为：

1. 逾期复习。
2. 今日复习。
3. 今日新词。

#### 单词卡片内容

- 单词。
- 音标。
- 释义或中文翻译。
- 当前任务类型。
- 当前阶段或盒位。
- 当前停留时长。
- 操作按钮：
  - 我会了。
  - 再学一次。
  - 先跳过。

#### 记录规则

每次提交学习结果时记录：

- `metaWordId`
- 学习动作：新学或复习。
- 学习结果：正确、错误、跳过。
- 总停留时长。
- 有效专注时长。
- 空闲时长。
- 交互次数。
- 注意力状态。

#### 与全局卡片盒同步

计划内学习结果写入 `StudyWordProgress` 后，应同步更新 `StudentWordMemory`：

- “我会了”：提升全局盒位，增加连续答对次数。
- “再学一次”：降低或保持低盒位，标记自动错词。
- “先跳过”：按错误处理，标记自动错词。

#### 验收标准

- 今日学习完成后首页任务状态实时更新。
- 学习计划完成率只由计划内任务决定。
- 每个学习结果同时保留计划内记录和全局记忆记录。
- 错误和跳过的词会出现在自动错词本。

### 3. 自由学习已分配词书

#### 功能说明

学生可以对已分配词书发起自由学习。自由学习不属于老师学习计划，不影响计划完成率、缺勤判断和老师布置任务的今日完成状态。

#### 入口

- 我的词书列表。
- 词书详情页。
- 单词搜索结果。
- 单词详情页。
- 今日任务完成后的“继续自由学习”入口。

#### 队列来源

自由学习队列仅从学生已分配词书中产生。推荐排序：

1. 全局卡片盒中到期复习的词。
2. 自动错词本中的高优先级词。
3. 手动收藏但未掌握的词。
4. 未学习过的新词。

#### 结果影响

- 写入 `StudentWordMemory`。
- 写入 `student_word_memory_events` 事件流水，来源标记为自由学习。
- 不写入 `StudyDayTask`。
- 不增加老师学习计划完成数。

#### 验收标准

- 未分配词书不能进入自由学习。
- 自由学习提交后，个人卡片盒状态更新。
- 自由学习不会改变老师计划的完成率。

### 4. 两层单词卡片盒

#### 计划内层

现有 `StudyWordProgress` 继续绑定 `studentStudyPlanId`，用于学习计划内的阶段、下次复习日期、正确次数、错误次数和掌握状态。

职责：

- 支撑老师发布的每日学习任务。
- 支撑计划完成率、缺勤、复习积压。
- 支撑老师查看学生在计划内的执行情况。

#### 全局层

新增 `StudentWordMemory`，维度为 `studentId + metaWordId`，用于学生长期个人记忆。

推荐字段：

| 字段 | 说明 |
|------|------|
| `id` | 主键 |
| `studentId` | 学生 ID |
| `metaWordId` | 单词 ID |
| `boxLevel` | 全局卡片盒位 |
| `masteryLevel` | 掌握度 |
| `nextReviewDate` | 下次个人复习日期 |
| `correctTimes` | 累计答对次数 |
| `wrongTimes` | 累计答错次数 |
| `correctStreak` | 连续答对次数 |
| `lastResult` | 最近一次结果 |
| `lastSource` | 最近来源：计划学习、自由学习、正式考试、自测 |
| `autoWrong` | 是否在自动错词本 |
| `favorite` | 是否手动收藏 |
| `lastStudiedAt` | 最近学习时间 |
| `createdAt` | 创建时间 |
| `updatedAt` | 更新时间 |

#### 卡片盒规则

- 新词默认进入低盒位。
- 答对升盒。
- 答错或跳过降盒或保持低盒位。
- 盒位越低，复习优先级越高。
- 不同盒位对应不同复习间隔，例如 `[0, 1, 2, 4, 7, 15, 30]`。
- 达到高盒位并连续答对后，可从自动错词本中移出，但仍保留全局记忆状态。

#### 验收标准

- 同一个学生对同一个单词只有一条全局卡片盒记录。
- 同一个单词出现在多个词书或计划中时，全局记忆状态可复用。
- 计划内进度和全局记忆状态互不替代。

### 5. 自动错词本与手动收藏夹

#### 自动错词本

自动错词本由学习和考试结果驱动，不需要学生手动加入。

进入条件：

- 今日学习中点击“再学一次”。
- 今日学习中点击“先跳过”。
- 自由学习中答错或跳过。
- 正式考试答错。
- 自测答错。

移出条件：

- 连续答对达到配置阈值。
- 全局盒位达到掌握阈值。
- 首期不允许学生手动移出自动错词本，避免把未掌握词隐藏；只能通过连续答对和升盒自动移出。

#### 手动收藏夹

学生可以在单词详情中收藏或取消收藏。

收藏夹用途：

- 个人重点词列表。
- 自由学习队列优先来源。
- 个人复习入口。

#### 展示规则

- 错词本和收藏夹分开展示。
- 同一个词可以同时是错词和收藏词。
- 列表支持按最近错误、错误次数、下次复习日期、盒位排序。

#### 验收标准

- 考试答错后对应词进入自动错词本。
- 学习中答错或跳过后对应词进入自动错词本。
- 学生收藏词后，收藏夹立即可见。
- 取消收藏不影响自动错词状态。

### 6. 考试与自测

#### 正式考试

正式考试由老师或管理员创建，学生只能作答分配给自己的考试。

规则：

- 正式考试成绩进入老师评价。
- 正式考试历史对学生本人可见。
- 正式考试答错同步更新全局卡片盒和自动错词本。

#### 自测练习

学生可从已分配词书发起自测。

规则：

- 自测只允许选择已分配词书。
- 自测可配置题量。
- 自测成绩仅进入学生个人统计。
- 自测答错同步更新全局卡片盒和自动错词本。
- 自测不进入老师正式考试完成率。

#### 数据建议

扩展 `Exam` 增加类型字段，本设计不新增独立自测表，以复用现有题目、提交和结果能力：

- `FORMAL`：正式考试。
- `SELF_PRACTICE`：学生自测。

#### 验收标准

- 学生不能为未分配词书创建自测。
- 自测结果不出现在老师正式考试评价统计中。
- 正式考试和自测结果都能进入学生个人历史。
- 答错题目会更新全局错词状态。

### 7. 学习日历与分析统计

#### 学习日历

展示最近 30 天每日状态：

- 已完成。
- 进行中。
- 未开始。
- 缺勤。
- 无计划但有自由学习。

#### 分析指标

- 连续学习天数。
- 累计完成词数。
- 今日完成率。
- 近 30 天正确率趋势。
- 近 30 天注意力趋势。
- 自动错词数量变化。
- 复习积压趋势。
- 平均有效停留时长。
- 长停留次数。
- 空闲中断次数。

#### 数据来源

- `StudyDayTask`
- `StudyRecord`
- `StudentAttentionDailyStat`
- `StudyWordProgress`
- `StudentWordMemory`
- 正式考试和自测结果

#### 验收标准

- 最近 30 天日历能区分计划学习和自由学习。
- 正确率趋势包含计划学习、自由学习和自测，可按来源筛选。
- 注意力趋势基于已有停留统计。
- 复习积压来自到期且未完成的卡片盒和计划任务。

### 8. 个人中心

#### 可编辑内容

学生可编辑：

- 显示名。
- 邮箱。
- 手机号。
- 密码。

#### 只读内容

学生只读：

- 用户名。
- 角色。
- 状态。
- 所属班级。
- 负责老师。
- 最近登录时间。

#### 密码修改规则

- 必须输入旧密码。
- 新密码满足长度和复杂度要求。
- 修改成功后立即清除当前登录态，要求学生使用新密码重新登录。

#### 验收标准

- 学生不能修改自己的角色、用户名、班级或老师归属。
- 学生资料更新后 `/api/auth/me` 返回最新展示信息。
- 密码修改后旧密码无法登录。

### 9. 首页轻提醒

#### 提醒类型

- 今日学习任务未完成。
- 存在逾期复习。
- 存在待完成正式考试。
- 全局卡片盒中存在到期复习。

#### 实现原则

- 不新增消息中心。
- 不新增消息表。
- 首页请求时聚合当前任务、考试和卡片盒状态生成提醒。
- 提醒只表达当前状态，不承诺历史消息追踪。

#### 验收标准

- 提醒数量和首页任务状态一致。
- 学生完成任务或考试后，提醒消失或更新。
- 刷新页面后提醒可由后端状态重新计算。

## 推荐接口清单

### 首页

- `GET /api/students/me/dashboard`
  - 返回今日任务摘要、轻提醒、连续学习、待考试数量、卡片盒到期复习数量。

### 今日学习

保留现有接口：

- `GET /api/students/me/study-plans`
- `GET /api/students/me/study-plans/{id}/today`
- `POST /api/students/me/study-plans/{id}/records`
- `GET /api/students/me/study-plans/{id}/attention`

调整要求：

- 计划内学习记录提交成功后同步更新 `StudentWordMemory`。

### 自由学习

- `GET /api/students/me/dictionaries/{dictionaryId}/self-study/queue`
- `POST /api/students/me/dictionaries/{dictionaryId}/self-study/records`

要求：

- 校验词书已分配给当前学生。
- 只更新全局卡片盒和自由学习记录，不更新计划任务。

### 全局卡片盒

- `GET /api/students/me/word-memory`
- `GET /api/students/me/wrong-words`
- `GET /api/students/me/favorite-words`
- `PATCH /api/students/me/word-memory/{metaWordId}/favorite`

要求：

- 所有查询仅返回当前学生自己的数据。
- 收藏接口只允许操作已分配词书范围内出现过或可见的单词。

### 考试与自测

保留现有正式考试接口，并扩展：

- `POST /api/exams/self-practice`
- `GET /api/exams/history?type=FORMAL|SELF_PRACTICE`

要求：

- 自测创建仅允许当前学生本人。
- 自测词书必须已分配给当前学生。
- 提交考试后同步更新全局卡片盒。

### 学习统计

- `GET /api/students/me/stats/calendar?days=30`
- `GET /api/students/me/stats/trends?days=30`

要求：

- 支持按来源区分计划学习、自由学习、正式考试、自测。
- 默认返回最近 30 天。

### 个人中心

- `GET /api/students/me/profile`
- `PATCH /api/students/me/profile`
- `PATCH /api/students/me/password`

要求：

- 资料编辑只允许显示名、邮箱、手机号。
- 密码修改必须校验旧密码。

## 数据模型建议

### `student_word_memories`

```sql
CREATE TABLE student_word_memories (
    id BIGSERIAL PRIMARY KEY,
    student_id BIGINT NOT NULL,
    meta_word_id BIGINT NOT NULL,
    box_level INT NOT NULL DEFAULT 0,
    mastery_level NUMERIC(5, 2) NOT NULL DEFAULT 0,
    next_review_date DATE,
    correct_times INT NOT NULL DEFAULT 0,
    wrong_times INT NOT NULL DEFAULT 0,
    correct_streak INT NOT NULL DEFAULT 0,
    last_result VARCHAR(32),
    last_source VARCHAR(32),
    auto_wrong BOOLEAN NOT NULL DEFAULT FALSE,
    favorite BOOLEAN NOT NULL DEFAULT FALSE,
    last_studied_at TIMESTAMP,
    created_at TIMESTAMP,
    updated_at TIMESTAMP,
    CONSTRAINT uk_student_word_memory UNIQUE (student_id, meta_word_id)
);
```

### 考试类型

建议给 `exams` 增加：

- `exam_type VARCHAR(32) NOT NULL DEFAULT 'FORMAL'`
- `created_by_student_id BIGINT`

正式考试使用 `FORMAL`，学生自测使用 `SELF_PRACTICE`。

### 学习记录来源

新增 `student_word_memory_events` 作为全局卡片盒事件流水，记录所有会影响长期记忆状态的动作：

| 字段 | 说明 |
|------|------|
| `id` | 主键 |
| `student_word_memory_id` | 全局记忆记录 ID |
| `student_id` | 学生 ID |
| `meta_word_id` | 单词 ID |
| `source_type` | 来源：计划学习、自由学习、正式考试、自测 |
| `source_id` | 来源业务 ID，例如计划任务、考试或自学会话 |
| `dictionary_id` | 关联词书 ID |
| `result` | 本次结果 |
| `box_level_before` | 变更前盒位 |
| `box_level_after` | 变更后盒位 |
| `created_at` | 创建时间 |

## 分期建议

### 第一阶段：首页与卡片盒基础

- 学生首页聚合摘要。
- `StudentWordMemory` 模型。
- 计划内学习结果同步全局卡片盒。
- 自动错词本。
- 手动收藏夹。

### 第二阶段：自由学习与自测

- 已分配词书自由学习。
- 自由学习记录。
- 学生自测。
- 自测结果同步错词本。

### 第三阶段：统计与个人中心

- 最近 30 天学习日历。
- 分析型统计趋势。
- 学生资料编辑。
- 学生密码修改。

## 测试与验收

### 后端测试

- 学生只能访问自己的首页、卡片盒、错词本、收藏夹、统计和个人中心。
- 学生不能自由学习未分配词书。
- 计划内学习提交会更新 `StudyWordProgress` 和 `StudentWordMemory`。
- 自由学习提交不会更新 `StudyDayTask`。
- 正式考试和自测都能更新错词本。
- 自测不会进入老师正式考试统计。
- 密码修改必须校验旧密码。

### 前端测试

- 学生登录后默认展示今日任务首页。
- 无计划时展示已分配词书和自由学习入口。
- 学生看不到管理按钮。
- 错词本和收藏夹状态更新后立即反映到列表。
- 自测和正式考试有明确视觉区分。
- 学习日历正确展示最近 30 天状态。

### 回归测试

- 老师发布学习计划后，学生仍可正常看到今日任务。
- 现有考试提交和考试历史不因自测类型扩展而回归。
- 管理员仍跳转管理端。
- 教师和管理员现有管理功能不受学生端改造影响。
