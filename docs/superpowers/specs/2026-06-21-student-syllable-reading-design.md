# 学生端单词音节拆读融合设计

## 目标

在现有学生端逐词学习卡片中加入按音节拆读能力，帮助学生建立单词拼写、发音和整体词义之间的联系。功能应保持当前“今日任务中枢”的视觉语言，不另建页面，也不改变计划学习、自由学习和记忆盒位的既有流程。

## 使用范围

- 今日计划学习和自由学习共用同一套音节拆读组件。
- 当前原型为前端演示数据；后续接入接口时，由单词数据返回音节文本和音标。
- 自测与正式考试保持答题界面，不展示音节提示，避免泄露答案。

## 卡片布局

音节区常驻在单词音标与中文释义之间，顺序为：

1. 单词、整词音标与任务类型。
2. 彩色音节拆分行，例如 `re · sil · ient`。
3. 各音节按钮，展示拼写、对应音标和播放图标。
4. “整词朗读”和“慢速拼读”两个操作。
5. 中文释义、停留时间和学习结果按钮。

音节数量使用响应式网格排列。三音节在 390px 屏幕上一行展示；320px 窄屏允许自动换行，不出现横向滚动。

## 视觉规则

- 沿用现有蓝、锈红、绿色状态色，为相邻音节提供可区分但不过度鲜艳的颜色。
- 音节按钮使用 8px 圆角、细边框和白色表面，与现有学习卡一致。
- 当前播放音节使用对应色的浅色背景和实色播放图标，不引入额外插画、渐变或新字体。
- 播放按钮使用现有 Phosphor 图标库。

## 数据结构

每个学习词条增加：

```js
{
  word: 'resilient',
  phonetic: '/rɪˈzɪliənt/',
  definition: 'adj. 有弹性的；能恢复的；有韧性的；坚韧的',
  syllables: [
    { text: 're', phonetic: '/rɪ/' },
    { text: 'sil', phonetic: '/ˈzɪl/' },
    { text: 'ient', phonetic: '/iənt/' },
  ],
}
```

计划学习和自由学习直接读取当前词条，不维护第二份音节状态。

## 播放交互

- 点击单个音节：朗读该音节，并高亮当前音节。
- 点击“整词朗读”：以正常语速朗读完整单词。
- 点击“慢速拼读”：按顺序朗读各音节，最后朗读完整单词。
- 新的播放操作会取消前一次未完成的朗读，避免声音重叠。
- 播放期间显示当前状态文本；完成或取消后恢复默认提示。

原型使用浏览器 `speechSynthesis`。如果浏览器不支持语音能力，按钮仍提供短暂视觉高亮，并显示“当前浏览器不支持语音播放”，不阻断学习结果提交。

## 可访问性

- 每个音节按钮有完整可访问名称，例如“播放音节 re，发音 /rɪ/”。
- 播放状态区域使用 `aria-live="polite"`。
- 按钮保留键盘焦点样式，颜色之外同时依赖图标和状态文字表达播放状态。
- 尊重 `prefers-reduced-motion`，慢速拼读不依赖动画完成语义表达。

## 验收标准

- 今日学习和自由学习均展示当前单词的音节拆分。
- 点击任一音节会出现对应的当前播放状态。
- 整词朗读和慢速拼读按钮可操作，且不会与上一次播放重叠。
- 切换到下一词后，音节、音标和释义同步更新，播放状态清空。
- “我会了”“再学一次”“先跳过”原有行为不回归。
- 390x844 和 320x740 视口无横向溢出、文字遮挡或底部导航冲突。
- 构建通过，浏览器控制台无错误，设计 QA 最终结果为 `passed`。

## 非目标

- 不进行自动音节算法解析或词典级发音校准。
- 不上传或管理真人录音文件。
- 不在考试和自测题目中展示拆读提示。
- 不改变后端学习记录、计划完成率或全局卡片盒规则。
