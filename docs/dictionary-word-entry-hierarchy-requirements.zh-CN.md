# 词书词条层级需求说明

## 背景

当前的 `dictionary_words` 被建模为“词典”和 `meta_word` 之间的简单关联关系。

这已经无法满足新的需求：

1. 同一个单词在同一本书里可能出现多次。
2. 单词在书中的位置是层级化的，而不是一个固定的 `chapter` 字段就能表达。
3. 学习进度仍然可能按单词维度追踪，即使这个单词后来在书中再次出现。

因此，`dictionary_words` 应该被视为“书内词条出现记录”，而不只是唯一关联关系。

## 已确认的产品前提

以下前提基于当前讨论结果整理：

1. 同一个 `meta_word` 必须允许在同一本词书中出现多次。
2. 如果导入时发现同一个词在同一本书里重复出现，不能因为这个原因导致导入失败。
3. 书内位置必须支持任意层级，例如：
   - `第1章 > 第2节`
   - `第3单元 > 第1课`
   - `Module 2 > Lesson 4 > Vocabulary`
4. 在学习过程中，如果某个重复出现的单词已经学过，可以标记为已学习。
5. 系统仍然需要保留词条在书中的原始顺序。

## 为什么当前模型不够用

当前行为默认 `(dictionary_id, meta_word_id)` 只能有一条唯一关系。

这会带来几个问题：

1. 同一个单词在同一本词书中的重复出现无法录入。
2. 导入逻辑会把同词的多次出现静默合并成一条关系。
3. 词典浏览接口无法返回层级位置和书内顺序。
4. 学习计划的发词顺序基于插入顺序，而不是基于书的编排结构。
5. 学习进度目前按 `meta_word_id` 跟踪，因此后续章节里重复出现的词会被视为已学过。

## 功能调整范围

本节描述的是功能层面必须调整的内容，不涉及具体实现方案。

### 1. 数据含义调整

`dictionary_words` 应当表示“某个单词在某本词书中的一次出现”，而不只是唯一的词典-单词关联。

每一次出现至少需要承载以下信息：

1. 所属词典
2. 对应的 `meta_word`
3. 在书中的层级路径
4. 在该层级下稳定可排序的位置

### 2. 支持重复词条

系统必须允许同一个 `meta_word_id` 在同一个词典中出现多次，只要这些记录代表的是书中不同位置或不同出现次数。

功能要求：

1. 手工新增时必须允许同词重复出现。
2. 批量新增时，只要位置不同，就必须允许同词重复出现。
3. CSV/JSON 导入时，不能因为同词重复出现而报错或被静默丢弃。
4. 重复判断规则需要从“同一本词典里同一个词”调整为“同一本词典里同一个书内词条位置”。

### 3. 支持多层级位置

系统必须支持多层级书籍结构，而不是固定为 `chapter`、`section`、`lesson` 这样的字段。

功能要求：

1. 能支持一层、两层或更多层。
2. 层级名称必须是通用的，不能写死为某一种教材术语。
3. 排序时必须先按层级顺序，再按叶子节点中的词条顺序。
4. 接口返回结果必须能让前端完整展示书内路径。

### 4. 词典浏览能力

当调用方需要看到书内上下文时，词典单词浏览接口不能再只返回 `MetaWord` 信息。

必要调整：

1. 词典单词列表返回结果必须包含层级路径和书内顺序。
2. 词典内搜索结果必须能展示匹配词出现在书中的什么位置。
3. 如果一个词在同一本词典中出现多次，接口必须能明确暴露所有出现位置。
4. 排序必须稳定，并基于显式层级/顺序，而不是隐式插入顺序。

### 5. 单词详情中的关联辞书回显

单词详情需要新增“关联辞书回显”能力，用于展示当前单词出现在哪些辞书、以及各自位于辞书中的什么位置。

必要调整：

1. 单词详情接口必须支持返回该单词关联的辞书列表。
2. 对于每一个关联辞书，必须返回该单词在辞书中的层级路径信息。
3. 如果一个单词在同一本辞书中出现多次，必须能展示该辞书下的所有出现位置，而不是只显示一次。
4. 返回结果应支持前端直接展示类似“辞书名 + 层级路径”的信息，例如“高中核心词汇 > 第1单元 > 第2课”。
5. 如果当前用户对某些辞书没有查看权限，则回显结果必须遵循现有权限控制规则。

### 6. 导入体验

导入行为必须对录入人员友好，不能因为教材编排的复杂性给录入造成障碍。

必要调整：

1. 同一个词在不同单元、课时或小节里重复出现时，必须导入成功。
2. 导入输入格式必须支持层级位置和条目顺序。
3. 导入校验必须校验词条结构，而不只是单词元数据。
4. 导入结果统计应区分：
   - 复用已有的 meta word 数量
   - 新建的 meta word 数量
   - 新增的词典词条数量
   - 校验失败的行数

### 7. 学习计划行为

由于“书内词条”和“是否学过的单词”不再完全等价，学习行为需要明确产品语义。

当前已达成的方向：

1. 学习进度可以继续按 `meta_word` 维度追踪。
2. 当一个单词在后续章节再次出现时，可以显示为已学习。

必要调整：

1. 新词分配顺序应按书内层级顺序，而不是按插入顺序。
2. 重复词条不能阻塞导入，也不能从词典浏览中消失。
3. 如果学习流程走到一个重复出现的词条，系统应能识别其已学状态，而不是把它再次当成全新词。
4. 学习任务相关接口或界面，可能需要显示“该词条为重复出现且已学习”。

### 8. 进度与统计

进度统计口径必须重新定义，否则很容易产生误导。

原因：

1. “词典词条总数”和“唯一单词总数”不再相同。
2. 如果学习进度按唯一单词计算，但分母却使用词条总数，完成度会失真。

必要调整：

1. 明确区分“唯一单词指标”和“词条指标”。
2. 明确学习进度采用哪一种口径。
3. 确保重复词条不会让进度表现得过低或过高，造成理解偏差。

### 9. 考试与其他下游功能

所有依赖词典单词的下游功能，都需要明确它们是面向“唯一单词”还是“书内词条”工作的。

至少需要明确：

1. 考试题池到底按唯一词取，还是按书内词条取。
2. 所有词典级统计都要明确统计的是“唯一词数”还是“出现次数”。

## 开发前必须确认的产品决策

以下决策建议在正式开发前先确认。

### 开发前必须确认

1. 一个“唯一词典词条”到底如何定义：
   - 相同词典 + 相同层级路径 + 相同顺序
   - 或者其他规则
2. 如果两条记录路径和顺序都相同，是否仍应视为非法重复。
3. 层级顺序是否完全由导入数据提供，还是允许系统自动补齐。
4. 手工新增接口是否必须立即支持把词加到指定层级位置。
5. 词典搜索结果的表现形式是：
   - 每次出现都返回一条结果
   - 还是返回一个词，并附带多个出现位置
6. 单词详情页中的关联辞书回显，是按“辞书分组后列出多个位置”，还是“每次出现单独一条”。
7. 学习进度分母到底使用：
   - 唯一单词数
   - 词条数
   - 还是两种都展示
8. 学习任务中是否要显式向学生展示“这是重复出现的已学词”。

### 可以后置决定

1. 考试是否支持按层级节点筛选出题
2. 教师端浏览是否支持树形视图、平铺视图，还是两者都支持
3. 层级标签是否需要做归一化或多语言处理
4. 单词详情页中是否需要支持从关联辞书位置直接跳转到对应章节

## 建议的产品方向

为了让整体行为保持一致，建议采用以下方向：

1. 把 `dictionary_words` 视为“词典中的词条出现记录”，而不是唯一关系。
2. 允许同一个词在同一本词典中重复出现。
3. 用通用层级路径模型表达书内结构，而不是固定 `chapter/section/lesson` 字段。
4. 除非后续需求证明有必要，否则学习进度优先继续按唯一单词维度跟踪。
5. 对于重复出现的词条，在适当场景下显示为已学习，但不能从词典结构中丢失。
6. 所有面向用户的统计都应区分“唯一单词”与“书内词条”。

## 历史数据兼容预期

旧版模型下已有的词典数据需要一个兼容策略。

建议的兼容预期：

1. 历史数据应被视为合法的单层级词条。
2. 历史顺序应尽量保留。
3. 即使后续还没有回填更丰富的层级信息，旧数据也应保持可读、可用。
