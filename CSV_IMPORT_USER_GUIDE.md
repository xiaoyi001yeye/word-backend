# CSV文件导入功能用户手册

## 1. 功能概述

CSV文件导入功能允许用户将自定义的单词数据批量导入到指定的辞书中。该功能支持标准CSV格式文件，可以快速扩充您的个人词库。

## 2. 使用前提

### 2.1 系统要求
- 已登录系统并拥有创建辞书的权限
- 需要有一个用户创建的辞书（不能向系统导入的辞书中添加单词）
- 浏览器支持文件上传功能

### 2.2 文件准备
- 准备UTF-8编码的CSV文件
- 文件大小不超过10MB
- 文件扩展名为.csv

## 3. 操作步骤

### 3.1 访问导入功能
1. 登录系统后，在左侧辞书列表中找到目标辞书
2. 点击辞书卡片右上角的"+"按钮
3. 在弹出菜单中选择"导入CSV文件"

### 3.2 准备CSV文件
按照以下格式准备您的CSV文件：

#### 标准格式（推荐）
```
单词,音标,定义,词性,例句,翻译,难度
apple,/ˈæpəl/,n. 苹果,名词,I ate an apple.,我吃了一个苹果,2
book,/bʊk/,n. 书籍,名词,Read a book.,读一本书,2
beautiful,/ˈbjuː.tɪ.fəl/,adj. 美丽的,形容词,She is beautiful.,她很美丽,3
```

#### 最小格式（仅必需字段）
```
单词,定义
apple,n. 苹果，苹果树...
book,n. 书籍；卷；帐簿...
```

### 3.3 上传文件
1. 点击"选择文件"按钮
2. 选择准备好的CSV文件
3. 根据文件实际情况选择是否包含表头行
4. 点击"开始导入"按钮

### 3.4 查看导入结果
导入过程中会显示进度条，完成后会显示详细的结果统计：
- **总记录数**：文件中包含的单词总数
- **成功导入**：成功添加到辞书的单词数
- **已存在**：系统中已有相同单词的数量
- **导入失败**：因格式错误等原因未能导入的单词数

## 4. CSV格式详细说明

### 4.1 字段定义

| 字段名 | 是否必需 | 数据类型 | 说明 | 示例 |
|--------|----------|----------|------|------|
| 单词 | 是 | 文本 | 英文单词，1-100字符 | apple |
| 音标 | 否 | 文本 | 单词发音，建议使用国际音标 | /ˈæpəl/ |
| 定义 | 是 | 文本 | 单词含义说明 | n. 苹果 |
| 词性 | 否 | 文本 | 语法词性分类 | 名词 |
| 例句 | 否 | 文本 | 包含该单词的句子 | I like apples. |
| 翻译 | 否 | 文本 | 中文翻译 | 我喜欢苹果。 |
| 难度 | 否 | 数字 | 难度等级1-5，1最简单，5最难 | 2 |

### 4.2 格式要求
- **编码**：必须使用UTF-8编码
- **分隔符**：使用英文逗号(,)分隔各字段
- **换行符**：使用标准换行符(\n)
- **引号**：包含逗号的字段需用双引号包围
- **转义规则**：字段内的双引号需要用两个双引号表示，例如：`"He said, ""Hello world"""`

### 4.3 特殊字符处理
```
正确示例：
"hello,world",/həˈloʊ/,interjection. 用来打招呼,感叹词,"Hello, how are you?",你好，你怎么样？,1

错误示例：
hello,world,/həˈloʊ/,interjection. 用来打招呼,感叹词,Hello, how are you?,你好，你怎么样？,1
```

## 5. 常见问题解答

### 5.1 文件上传失败
**问题现象**：点击上传后提示文件过大或格式错误

**解决方案**：
- 检查文件大小是否超过10MB
- 确认文件扩展名为.csv
- 验证文件编码为UTF-8

### 5.2 导入结果显示失败
**问题现象**：导入完成后失败数量较多

**可能原因及解决方法**：
1. **格式错误**：检查CSV文件是否严格按照规定格式
2. **编码问题**：重新保存为UTF-8编码
3. **特殊字符**：检查是否有未转义的引号或逗号
4. **字段缺失**：确认必需字段（单词、定义）不为空

### 5.3 重复单词处理
**问题现象**：导入后发现有些单词没有新增

**说明**：
- 系统会自动检测重复单词
- 重复的单词不会创建新的记录
- 已存在的单词信息会被保留

### 5.4 导入速度慢
**问题现象**：大文件导入耗时较长

**优化建议**：
- 将大文件拆分为多个小文件分别导入
- 避免在网络高峰期进行导入操作
- 关闭其他占用网络资源的应用

## 6. 最佳实践建议

### 6.1 文件准备建议
1. **数据清洗**：导入前检查数据质量，去除重复项
2. **格式统一**：保持字段格式的一致性
3. **适度规模**：单次导入建议不超过5000条记录
4. **备份原始**：保留原始数据文件作为备份

### 6.2 导入策略
1. **分批导入**：大词库可分多次导入
2. **测试先行**：先用少量数据测试格式正确性
3. **及时验证**：导入后立即检查结果准确性
4. **记录管理**：建立导入记录便于后续维护

### 6.3 数据质量控制
1. **标准化**：统一难度评级标准
2. **完整性**：尽量填写所有可用字段
3. **准确性**：确保定义和翻译的准确性
4. **时效性**：定期更新和维护词库内容
